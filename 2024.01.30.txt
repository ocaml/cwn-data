        values that denote functions mapping points of the
        cartesian plane to colors and combinators are provided to
        define and compose them.

        Renderers for PDF, SVG, Cairo and the HTML canvas are
        distributed with the library. An API allows to implement
        new renderers.

        Vg is distributed under the ISC license.

  library and the `vg.svg' library is deprecated. The `Vgr_htmlc' canvas
  renderer has been rewritten with [brr] which becomes an optional
  dependency of the package while the dependencies on `js_of_ocaml' and
  `js_of_ocaml-ppx' are dropped for good.

  Ten years ago neither dark themes nor `.mld' files did exist so the
  [tutorial], the [semantics] and the [image howto] have been slightly
  reworked and moved out of the Vg module documentation to their own

  As indicated above there could be options to eliminate two existing

  2. TLS support for Async, EIO, and LWT could instead be merged as
     sublibraries of `caqti-async', `caqti-eio', and `caqti-lwt', and
     `caqti-tls' could be merged into `caqti'. The main reason for not

     `ssl' library should we want to support it, since it is tied to the
     network stack.

  While switching to the second option is simple, switching to the third
  option would require more work on my part, delaying the release, but I
  could be convinced if we feel strongly for the end result.

  The `caqti-tls' package is (currently) only needed to define a shared
  configuration key which depends on the `tls' library.
