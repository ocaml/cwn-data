* YAMLx: pure-OCaml YAML 1.2/1.1 library
:PROPERTIES:
:CUSTOM_ID: 3
:END:
Archive: https://discuss.ocaml.org/t/ann-yamlx-pure-ocaml-yaml-1-2-1-1-library/17970/1

** Martin Jambon announced


Hello,

I'm excited to announce the availability of [[https://github.com/mjambon/yamlx][YAMLx]], a complete implementation of the YAML 1.2 and 1.1 standards. It is intended to support most uses of YAML including its use as a configuration file format in new applications and analyzing YAML files of other applications.

For a fuller description, check out the [[https://github.com/mjambon/yamlx/releases/tag/0.1.0][release notes]].

Get started with ~opam install yamlx~. Then use the ~yamlx~ companion command to check that it parses and interprets your favorite YAML files correctly.

Here's an example showing the use of the library to convert YAML to JSON:

#+begin_src ocaml
(* Read YAML from stdin with YAMLx, print JSON to stdout with Yojson

   This is a demo showing how to use the YAMLx library with the default
   settings. 'YAMLx.Values.one_of_yaml_exn' offers options to restrict
   the input language or to force the interpretation as YAML 1.1 instead of
   YAML 1.2.

   Compile with:
     ocamlfind opt -o yaml-to-json \
       -package yamlx,yojson -linkpkg yaml_to_json.ml
*)

open Printf

let rec yojson_of_yamlx (x : YAMLx.value) : Yojson.Safe.t =
  match x with
  | Null _ -> `Null
  | Bool (_, x) -> `Bool x
  | Int (_, x) -> `Intlit (Int64.to_string x)
  | Float (_, x) -> `Float x
  | String (_, x) -> `String x
  | Seq (_, xs) -> `List (List.map yojson_of_yamlx xs)
  | Map (_, xs) ->
      `Assoc (List.map (fun (loc, k, v) ->
        match (k : YAMLx.value) with
        | String (_, k) -> (k, yojson_of_yamlx v)
        | _ ->
            ksprintf failwith "%s: only string keys are supported"
              (YAMLx.default_format_loc loc)
      ) xs)

let () =
  YAMLx.register_exception_printers ();
  stdin
  |> In_channel.input_all
  |> YAMLx.Values.one_of_yaml_exn
  |> yojson_of_yamlx
  |> Yojson.Safe.pretty_to_channel stdout
  |> print_newline
#+end_src

*** Funding

YAMLx is currently released under the *AGPL*. There is an ongoing fundraiser: once a funding goal is reached, the license will switch to the permissive *ISC* license for everyone. Donors above a certain threshold receive an immediate *commercial license*. See [[https://github.com/sponsors/mjambon][my GitHub Sponsors profile]] for details.
      



* ocaml-letsencrypt, an OCaml way to get TLS certificates
:PROPERTIES:
:CUSTOM_ID: 4
:END:
Archive: https://discuss.ocaml.org/t/ann-ocaml-letsencrypt-an-ocaml-way-to-get-tls-certificates/17975/1

** Calascibetta Romain announced


I am delighted to announce the release of [[https://github.com/robur-coop/ocaml-letsencrypt][ocaml-letsencrypt]] (version 2.1.0). This version introduces a new API that allows it to operate independently of a scheduler and an HTTP client. It is also a rewrite using our brand-new [[https://github.com/robur-coop/jws][jws]] library, which now utilises the [[https://erratique.ch/logiciel/jsont][jsont]] library (rather than [[https://github.com/ocaml-community/yojson][yojson]]).

~ocaml-letsencrypt~ implements [[https://letsencrypt.org/fr/docs/challenge-types/][challenges]] needed to obtain a certificate via the three ways: DNS, HTTP and ACME-TLS. It therefore enables a website deployment strategy (implemented with [[https://github.com/robur-coop/vif][Vif]], for example) to be provided entirely in OCaml.

We use it within our cooperative for projects such as:
- [[https://github.com/robur-coop/dns-letsencrypt-secondary/][~dns-letsencrypt-secondary~]], a unikernel that acts as a secondary DNS server capable of performing the DNS challenge for a specific domain and generating the certificate and private key. It generally works in tandem with our [[https://github.com/robur-coop/dns-primary-git][~primary-git~]], our primary DNS server.
- [[https://github.com/dinosaure/contruno][~contruno~]], a unikernel that acts as a TLS reverse proxy and is capable of performing HTTP challenges. The latter is currently being completely rewritten using [[https://discuss.ocaml.org/t/ann-mnet-a-new-tcp-ip-stack-for-unikernels-in-ocaml/17851][~mnet~]].

Good luck with deploying your websites, and happy hacking! Here’s hoping that ~jws~ also finds its place as a library within the OCaml ecosystem.

* I18n 5.0 — Internationalisation made easy for any OCaml app
:PROPERTIES:
:CUSTOM_ID: 7
:END:
Archive: https://discuss.ocaml.org/t/ann-i18n-5-0-internationalisation-made-easy-for-any-ocaml-app/17981/1

** Vincent Balat announced


The Ocsigen team is happy to announce the release of *ocsigen-i18n 5.0*, a small but practical internationalisation library for OCaml.

The big change in this release: ocsigen-i18n is no longer tied to Eliom. It can now be used in *any OCaml project*, with optional extensions for Tyxml and Eliom when you need them.

Ocsigen-i18n was initially written by @sagotch from Be Sport. This release has been made possible thanks to the work of [[https://github.com/mhjd][Habib]], funded by [[https://www.irill.org][IRILL]].

#+begin_quote
opam install ocsigen-i18n
#+end_quote

*** How it works

Translations are written in a plain *TSV file* (one key per line, one column per language):

#+begin_example
foo                 This is a simple key.        Ceci est une clé toute simple.
a_human             a human                      un humain
bar                 I am {{x}}.                  Je suis {{x}}.
baz                 There {{{c?are||is an}}} apple{{{c?s||}}} here!  Il y a {{{c?des||une}}} pomme{{{c?s||}}} ici !
bu                  I am {{x %s}} ({{n %d}}).    Je suis {{x %s}} ({{n %d}}).
#+end_example

The mini-templating language supports:

- ~{{x}}~ a string variable ~~x~
- ~{{x %d}}~ a typed variable using the given format specifier
- ~{{{c?yes||no}}}~ an optional boolean ~?c~ switching between two strings

Then in your code, a *PPX extension* turns ~[%i18n key]~ into the right call:

#+begin_src ocaml
print_endline [%i18n foo];
print_endline [%i18n bar ~x:[%i18n a_human]];
print_endline [%i18n baz ~c:(nb > 1)];
print_endline [%i18n bu ~x:"Jean-Michel" ~n:42];

(* Switching language explicitly *)
print_endline [%i18n foo ~lang:My_i18n.Fr];
#+end_src

The current language is held in a mutable reference you can swap (or replace with an Eliom scoped reference if you need per-session/tab languages).

*** Dune integration

The tool plugs into Dune very naturally. Generate the OCaml module from your TSV file with a rule:

#+begin_src lisp
(rule
 (target example_i18n.ml)
 (deps example_i18n.tsv)
 (action
  (run %{bin:ocsigen-i18n} --languages en,fr --input-file %{deps}
                           --output-file %{target})))
#+end_src

And wire the PPX in your library/executable:

#+begin_src lisp
(preprocess (pps ocsigen-i18n -- --default-module Example_i18n))
#+end_src

That’s all you need for a plain OCaml project.

*** Tyxml support

Pass ~--tyxml~ to the generator and the same ~[%i18n key]~ expression now produces a *list of Tyxml HTML elements* instead of a string:

#+begin_src ocaml
(* Builds an HTML fragment, ready to drop into a Tyxml tree *)
let greeting = [%i18n bar ~x:[%i18n a_human]]
#+end_src

Variables can themselves be lists of HTML nodes, so you can mix translated text with markup naturally:

#+begin_src ocaml
[%i18n bar ~x:[ txt "Jean-Michel ("
              ; txt (string_of_int id)
              ; txt ")" ]]
#+end_src

If you need a plain string in Tyxml mode (for an attribute, for instance), just prefix with ~S.~:

#+begin_src ocaml
[%i18n S.bar ~x:[%i18n S.a_human]]   (* string output *)
#+end_src

*** Eliom support

For client–server Eliom apps, pass ~--eliom~. The generator emits an ~.eliom~ file (so the same translations are available on both sides), implies ~--tyxml~, and adds ~[@@deriving json]~ on the language type so you can serialise it across the wire:

#+begin_src lisp
(rule
 (target example_i18n.eliom)
 (deps example_i18n.tsv)
 (action
  (run %{bin:ocsigen-i18n} --eliom --languages en,fr --input-file %{deps}
                           --output-file %{target})))
#+end_src

*** Multiple TSV files

You can split translations across several files. The PPX uses your module path to find the right one:

#+begin_src ocaml
[%i18n foo]                          (* default module *)
[%i18n MyI18n.foo]                   (* MyI18n.foo *)
[%i18n MyI18n.S.bar ~x:[%i18n S.foo]]
#+end_src

Optional ~--prefix~ / ~--suffix~ flags let you keep call-sites concise (~[%i18n Feature.foo]~ → ~Pr_Feature_i18n.foo ()~).

* ATD 4.1.0 - now with YAML support
:PROPERTIES:
:CUSTOM_ID: 8
:END:
Archive: https://discuss.ocaml.org/t/ann-atd-4-1-0-now-with-yaml-support/17982/1

** Martin Jambon announced


[[https://github.com/ahrefs/atd][ATD]] now has OCaml support for conveniently reading data from YAML files as well as from other formats with minimal effort. It means:

1. You define a [[https://github.com/ahrefs/atd/blob/master/atd-yamlx/examples/app_config.atd][type for your config data]].
2. ~atdml --mode jsonlike app_config.atd~ generates the OCaml code that converts a JSON-like AST containing source locations into your config data structure.
3.  [[https://github.com/ahrefs/atd/blob/master/atd-yamlx/examples/main.ml][Your program]] loads a YAML file, converts it to a JSON-like tree, and then calls the generated function to translate it into an OCaml data structure.
4. Your application prints useful error messages if the config is malformed or incomplete.

An [[https://github.com/ahrefs/atd/blob/master/atd-yamlx/examples][example]] is included in the distribution. It uses the [[https://discuss.ocaml.org/t/ann-yamlx-pure-ocaml-yaml-1-2-1-1-library/17970][recently released YAMLx library]].

_Note that YAMLx needs generous sponsors to fund the work that was done and [[https://github.com/mjambon/yamlx/blob/main/FUNDING.md][unlock a permissive ISC license]] for everyone. Or you can accept the terms of the [[https://en.wikipedia.org/wiki/GNU_Affero_General_Public_License][AGPL license]] if you're happy to share your application's source code with the users._

Other input file formats or other YAML libraries can be used similarly with minimum effort as long as source locations are available. For this, you'll need [[https://github.com/ahrefs/atd/blob/master/atd-yamlx/src/Atd_yamlx.ml][a bridge]] that converts your AST into the [[https://github.com/ahrefs/atd/blob/master/atd-jsonlike/src/AST.mli][JSON-like AST]].
